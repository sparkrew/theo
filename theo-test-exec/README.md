# Theo dynamic analyser

Dynamic analyser can record the execution of a given project. Currently, it only supports test execution monitoring.  

## How it works

Dynamic analyser uses the [Java Flight Recorder (JFR)](https://openjdk.org/jeps/328) to capture runtime data. It can record 10 types
of accesses or capabilities.

- ClassLoad
- Deserialization
- FileForce
- FileRead
- FileWrite
- Native Library Access
- Network Access
- Process Start
- Socket Read
- Socket Write

When one of these events is observed by the JFR, the monitor will get its stack trace and check if it has been called by a
third party library. If so, it will record it.

It also records events generated by the Theo java agent. Those events cover the sensitive APIs which are not detected by the 
JFR. 

## Limitations

- The processStart event cannot capture the exact sensitive API that was called.
- Any event that involves core JDK classes (e.g., `java.lang.*`) that are captured by the JFR cannot be captured by the Java agent either. On the bright side, so far, we have not found any such events. All core JDK sensitive APIS are currently captured by the JFR.
